# 桥梁设备全生命周期管理系统

> 华东交通大学软件学院 2026 届毕业设计
> 作者：程国忠 ｜ 指导教师：向华萍

## 📦 项目说明

本系统针对桥梁设备运维管理中存在的"重建轻管、数据孤岛、报废决策主观"等核心痛点，构建了一套覆盖**采购→投运→保养→维修→鉴定→报废**全生命周期的数字化管理平台。

系统基于 **TCO（Total Cost of Ownership，全寿命周期成本）模型**，结合气候系数 α 与交通量系数 β 的双因素动态调参机制，为设备的"修"与"换"提供量化决策依据。

## 🚀 快速开始

**方式一：本地浏览器双击打开**

直接双击 `index.html`，使用现代浏览器（Chrome / Edge / Firefox 最新版）打开即可。系统首次启动会自动初始化 IndexedDB 数据库并写入种子数据。

**方式二：本地静态服务器（推荐）**

```bash
# Python 3
cd 项目目录
python3 -m http.server 8080

# 或 Node.js
npx serve .
```

然后浏览器访问 `http://localhost:8080`

> ⚠ 系统依赖外部 CDN（React / Tailwind / ECharts / Babel），首次加载需要联网。

## 📊 核心功能模块

1. **登录与权限**（多角色 RBAC）
2. **首页仪表盘** — 8 项关键率指标 + 30 天费用趋势 + 待办事项
3. **全生命周期视图** — 设备时间轴：采购→投运→保养→维修→报废
4. **桥梁基础参数** — α 气候系数 + β AADT 系数 + 历史趋势
5. **设备档案管理** — 11 类设备 / 自动编码 / 保养超期提示
6. **保养计划** — α × β 动态周期推算 + 上报登记
7. **故障维修工单** — 申报→处理→关闭闭环 + MTTR / MTBF 统计
8. **采购入库** — 订单流转：待发货 → 运输中 → 已入库
9. **报废技术鉴定** — TCO 三栏决策面板（论文核心算法）
10. **统计分析** — 4 张图表：分类分布 / 费用趋势 / 故障排行 / TCO 对比
11. **阈值参数配置** — 3 滑块动态配置 TCO 关键参数
12. **用户与角色管理** — 增删改查 / 密码重置 / 状态切换
13. **审计日志** — 所有写操作可追溯

## 🎯 针对答辩意见的改进点

向华萍老师审查后提出 4 点关键意见，本版本针对性改进：

| 老师意见 | 改进方案 | 涉及模块 |
|---------|---------|---------|
| ① 缺登录界面 | 完整登录页 + 多角色 RBAC + 冻结账户检测 + 会话恢复 | LoginPage / AuthContext |
| ② 没体现全生命周期 | 新增"全生命周期"独立菜单，时间轴展示完整事件链 + 累计 TCO | LifecyclePage / lifecycle_event 表 |
| ③ 数据库没做好 | 17 张表严格对应论文 4.3 节物理表设计，含审计/事件/历史 | dbApi / 17 个 ObjectStore |
| ④ 统计率要真实算出来 | 8 项关键率全部由数据库聚合，无任何硬编码假数据 | services.statsOverview() |
