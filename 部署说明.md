# 部署说明

## 🚀 部署架构

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   Vue 3 前端     │────▶│  Spring Boot    │────▶│    MySQL 8      │
│   (Nginx/静态)   │     │   REST API      │     │   数据库        │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

## 📦 本地开发环境

### 前端启动

```bash
cd frontend
npm install           # 安装依赖
npm run dev           # 启动开发服务器 (http://localhost:5173)
```

### 后端启动

```bash
cd backend
mvn clean install     # 编译项目
mvn spring-boot:run   # 启动后端 (http://localhost:8080)
```

### 数据库初始化

```sql
-- 创建数据库
CREATE DATABASE bridge_lifecycle CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 执行建表脚本
SOURCE backend/src/main/resources/schema.sql;

-- 执行种子数据脚本
SOURCE backend/src/main/resources/data.sql;
```

### 配置文件修改

修改 `backend/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/bridge_lifecycle
    username: root
    password: your_password  # 修改为你的 MySQL 密码
```

## 🌐 生产环境部署

### 前端构建

```bash
cd frontend
npm run build          # 生成 dist 目录
```

构建产物在 `frontend/dist/`，包含：
- `index.html`
- `assets/*.js` (压缩后的 JS)
- `assets/*.css` (压缩后的 CSS)

### Nginx 配置

```nginx
server {
    listen 80;
    server_name bridge.example.com;

    # 前端静态文件
    location / {
        root /var/www/bridge/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 反向代理
    location /api/ {
        proxy_pass http://localhost:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 后端打包

```bash
cd backend
mvn clean package -Dmaven.test.skip=true
```

生成 `target/bridge-backend-1.0.0.jar`

### 后端启动命令

```bash
java -jar bridge-backend-1.0.0.jar \
  --spring.datasource.url=jdbc:mysql://prod-db:3306/bridge_lifecycle \
  --spring.datasource.username=prod_user \
  --spring.datasource.password=prod_password
```

## 🐧 Linux 服务器部署

### 系统要求

| 软件 | 版本 |
|------|------|
| JDK | 17+ |
| MySQL | 8.0+ |
| Nginx | 1.20+ |
| Node.js | 18+ (构建前端) |

### 部署步骤

1. **安装依赖**

```bash
# CentOS/RHEL
yum install -y java-17-openjdk mysql-server nginx

# Ubuntu/Debian
apt install -y openjdk-17-jdk mysql-server nginx
```

2. **配置 MySQL**

```bash
mysql -u root -p
CREATE DATABASE bridge_lifecycle CHARACTER SET utf8mb4;
CREATE USER 'bridge'@'localhost' IDENTIFIED BY 'bridge_password';
GRANT ALL PRIVILEGES ON bridge_lifecycle.* TO 'bridge'@'localhost';
FLUSH PRIVILEGES;
```

3. **部署后端**

```bash
# 上传 jar 包
scp backend/target/bridge-backend-1.0.0.jar user@server:/opt/bridge/

# 创建 systemd 服务
cat > /etc/systemd/system/bridge-backend.service << EOF
[Unit]
Description=Bridge Lifecycle Backend
After=mysql.service

[Service]
Type=simple
User=bridge
ExecStart=/usr/bin/java -jar /opt/bridge/bridge-backend-1.0.0.jar
Restart=always

[Install]
WantedBy=multi-user.target
EOF

systemctl enable bridge-backend
systemctl start bridge-backend
```

4. **部署前端**

```bash
# 上传构建产物
scp -r frontend/dist/* user@server:/var/www/bridge/

# 配置 Nginx
cp nginx.conf /etc/nginx/conf.d/bridge.conf
nginx -t
systemctl restart nginx
```

## 🔒 安全配置

### 后端安全

1. **修改默认密码**

执行 SQL 更改 admin 密码：
```sql
UPDATE sys_user SET password = '$2a$10$new_hash' WHERE username = 'admin';
```

2. **配置 CORS**

`application.yml`：
```yaml
cors:
  allowed-origins: https://bridge.example.com
  allowed-methods: GET,POST,PUT,DELETE
```

3. **JWT 密钥配置**

```yaml
jwt:
  secret: your-256-bit-secret-key-here
  expiration: 86400000  # 24小时
```

### 数据库安全

```sql
-- 删除空密码账户
DELETE FROM mysql.user WHERE password = '';

-- 禁止远程 root 登录
DELETE FROM mysql.user WHERE user = 'root' AND host != 'localhost';
```


## 🔍 常见问题

### Q: 前端请求后端跨域报错

A: 确保后端 CORS 配置正确，或使用 Nginx 反向代理统一域名。

### Q: 数据库连接失败

A: 检查 MySQL 服务状态、用户权限、防火墙端口 3306。

```bash
systemctl status mysql
mysql -u bridge -p -h localhost bridge_lifecycle
```

### Q: JWT Token 过期

A: 默认 24 小时过期，重新登录获取新 Token。可在 `application.yml` 调整过期时间。

### Q: 前端构建报错

A: 确保 Node.js 版本 ≥ 18，删除 `node_modules` 重装依赖：

```bash
rm -rf frontend/node_modules frontend/package-lock.json
npm install
```
