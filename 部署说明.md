# 部署说明

## 🚀 三种部署方式

### 方式一：本地双击运行（最简单）

直接双击 `index.html`，浏览器会自动打开。

**优点**：零配置
**缺点**：file:// 协议下部分浏览器（如 Chrome 严格模式）可能限制 IndexedDB；建议使用方式二

### 方式二：本地静态服务器（推荐）

```bash
# 进入项目目录
cd 桥梁设备全生命周期管理系统

# 方法 A：Python（系统自带）
python3 -m http.server 8080

# 方法 B：Node.js
npx serve .

# 方法 C：VS Code Live Server 插件
```

然后浏览器访问 `http://localhost:8080`

### 方式三：部署到服务器

将 `index.html` 上传到任何静态网站托管服务即可：

- Nginx / Apache
- GitHub Pages
- Vercel / Netlify
- 阿里云 OSS / 腾讯云 COS

> 示例 Nginx 配置：
> ```nginx
> server {
>   listen 80;
>   server_name bridge.example.com;
>   root /var/www/bridge;
>   index index.html;
> }
> ```

## 🔧 离线部署方案

如果答辩教室无法联网，可以预先下载所有 CDN 资源到本地：

1. 访问 CDN 链接，把以下文件下载到 `vendor/` 目录：
   - `react.production.min.js`
   - `react-dom.production.min.js`
   - `babel.min.js`
   - `echarts.min.js`
   - `tailwind.min.css`

2. 修改 `index.html` 顶部的 `<script>` 和 `<link>` 标签：
   ```html
   <script src="vendor/react.production.min.js"></script>
   <script src="vendor/react-dom.production.min.js"></script>
   <script src="vendor/babel.min.js"></script>
   <script src="vendor/echarts.min.js"></script>
   <link rel="stylesheet" href="vendor/tailwind.min.css">
   ```

3. 启动本地服务器即可离线运行。

## 🔍 常见问题

### Q: 打开后页面空白，控制台报错

A: 检查浏览器控制台（F12）：
- 如果是 CDN 加载失败 → 检查网络连接，或使用离线方案
- 如果是 React 报错 → 清除浏览器缓存后重试
- 如果是 IndexedDB 报错 → 浏览器禁用了存储，请允许网站使用 Cookie 与本地存储

### Q: 数据消失了 / 页面卡在加载中

A: 浏览器开发者工具 → Application → IndexedDB → 删除 `bridge_lifecycle_db` → 刷新页面，会重新初始化种子数据。

### Q: 隐身模式下无法保存数据

A: 隐身模式会限制 IndexedDB 容量，建议在普通窗口使用。

### Q: 演示前如何重置为初始数据？

A: 浏览器控制台执行：
```javascript
indexedDB.deleteDatabase('bridge_lifecycle_db');
location.reload();
```

或者在浏览器中：F12 → Application → Storage → Clear site data → 刷新

### Q: 如何修改种子数据？

A: 编辑 `index.html` 中的 `seedDataIfEmpty()` 函数，修改后清空数据库重新加载即可。

### Q: 答辩电脑无法预装 Chrome 怎么办？

A:
1. 优先使用便携版 Chrome（U 盘携带）
2. 退而求其次用 Edge（Windows 10/11 自带）
3. 保底方案：录制演示视频作为备份

## 📞 故障排查

打开 `index.html` 后按 F12 打开开发者工具：

1. **Console 面板**：查看 JavaScript 错误
2. **Network 面板**：检查 CDN 资源是否加载成功
3. **Application 面板** → IndexedDB：检查 `bridge_lifecycle_db` 是否存在 17 个 ObjectStore

如果还是无法解决，请联系 Claude 或检查浏览器是否更新到最新版本。
