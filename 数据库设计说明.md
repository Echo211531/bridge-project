# 数据库设计说明

> 本系统使用浏览器 IndexedDB 模拟 MySQL 8，包含 17 张 ObjectStore，对应论文 4.3 节物理表设计。

## 📚 表结构总览

| 编号 | 表名 | 中文名 | 主要用途 |
|------|------|-------|---------|
| 1 | sys_user | 系统用户 | 账号、密码、角色、状态 |
| 2 | sys_role | 系统角色 | 角色定义 |
| 3 | sys_role_menu | 角色-菜单 | 权限映射 |
| 4 | sys_config | 系统配置 | TCO 阈值参数 |
| 5 | sys_audit_log | 审计日志 | 写操作流水 |
| 6 | bridge | 桥梁基础参数 | α 气候系数、AADT |
| 7 | bridge_aadt_history | AADT 历史 | 交通量变化 |
| 8 | monitoring_record | 监测推送 | IoT 数据 |
| 9 | device_archive | 设备档案 | 设备主表 |
| 10 | device_category | 设备分类 | 11 类设备字典 |
| 11 | maintain_plan | 保养计划 | 周期、下次保养日期 |
| 12 | maintain_record | 保养记录 | 保养执行流水 |
| 13 | fault_order | 故障工单 | 故障申报与处理 |
| 14 | purchase_order | 采购订单 | 采购流程 |
| 15 | scrap_decision | 报废鉴定 | TCO 决策结论 |
| 16 | scrap_approval | 报废审批 | 二级审批工作流 |
| 17 | lifecycle_event | 生命周期事件 | 全周期事件流水 |

## 📋 核心表字段定义

### 1. sys_user（用户表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string (PK) | 用户 ID |
| username | string (UK) | 登录账号 |
| password | string | 密码哈希 |
| real_name | string | 真实姓名 |
| role_code | string | 角色代码 |
| dept | string | 所属部门 |
| phone | string | 手机号 |
| status | int | 1=正常, 0=禁用 |
| last_login_at | datetime | 最后登录时间 |
| create_time | datetime | 创建时间 |

### 2. bridge（桥梁基础参数表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string (PK) | 桥梁 ID |
| bridge_code | string | 桥梁编码 |
| bridge_name | string | 桥梁名称 |
| location | string | 所在位置 |
| build_year | int | 建成年份 |
| length_m | int | 桥长（米） |
| width_m | float | 桥宽（米） |
| span_m | int | 主跨（米） |
| design_load | string | 设计荷载 |
| climate_zone | string | 气候带代码 |
| alpha_coef | float | α 气候系数 |
| current_aadt | int | 当前 AADT |
| beta_coef | float | β AADT 系数 |
| aadt_update_at | date | AADT 更新日期 |

**α 气候系数取值：**
- 寒冷地区（cold）：1.30
- 严寒地区（severe_cold）：1.20
- 温和地区（mild）：1.00
- 湿热地区（humid_hot）：1.15
- 沿海地区（coastal）：1.25

**β AADT 系数阶梯：**
- AADT < 10000 → 0.85
- 10000 ≤ AADT < 30000 → 1.00
- 30000 ≤ AADT < 60000 → 1.15
- 60000 ≤ AADT < 100000 → 1.30
- AADT ≥ 100000 → 1.50

### 3. device_archive（设备档案表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string (PK) | 设备 ID |
| device_code | string (UK) | 设备编码（自动生成） |
| device_name | string | 设备名称 |
| category | string | 分类代码 |
| bridge_id | string (FK) | 所属桥梁 |
| install_position | string | 安装位置 |
| supplier | string | 供应商 |
| purchase_price | decimal | 购置价 |
| purchase_date | date | 采购日期 |
| commission_date | date | 投运日期 |
| design_life_y | int | 设计寿命（年） |
| status | string | 状态 |
| annual_maintain_cost | decimal | 年保养费 |
| annual_repair_cost | decimal | 年维修费 |
| fault_count_month | int | 本月故障次数 |

**设备状态枚举：**
- in_use：使用中
- in_stock：库存中
- maintaining：保养中
- fault：故障中
- scrapped：已报废
- disabled：停用

### 4. lifecycle_event（生命周期事件表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string (PK) | 事件 ID |
| device_id | string (FK) | 设备 ID |
| event_type | string | 事件类型 |
| title | string | 标题 |
| description | string | 描述 |
| operator | string | 操作人 |
| cost | decimal | 关联费用 |
| event_time | datetime | 发生时间 |

**事件类型枚举：**
- purchase：采购入库
- commission：投入运行
- maintain：保养
- repair：维修
- inspect：检查
- scrap：报废

### 5. scrap_decision（报废鉴定表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string (PK) | 鉴定 ID |
| device_id | string (FK) | 设备 ID |
| decision_date | date | 鉴定日期 |
| decision_user | string | 鉴定人 |
| tco_repair | decimal | 维修方案 TCO |
| tco_replace | decimal | 更换方案 TCO |
| residual_value | decimal | 残值估算 |
| target_years | int | 服役目标年限 |
| repair_estimate | decimal | 修复费用估算 |
| conclusion | text | 鉴定结论 |
| recommendation | string | 建议（repair/replace） |
| status | string | 审批状态 |

### 6. sys_config（系统配置表）

| Key | 默认值 | 取值范围 | 说明 |
|-----|-------|---------|------|
| repair_residual_threshold | 0.60 | 0.30–0.80 | 修复成本/残值阈值 |
| life_warning_ratio | 0.85 | 0.60–0.95 | 寿命警戒阈值 |
| market_discount_ratio | 0.80 | 0.50–1.00 | 市场折扣系数 |

## 🌱 种子数据规模

系统首次启动会自动写入：

- **8 个用户账号**（5 角色 + 1 冻结测试账号）
- **5 角色 + 12 菜单 + 角色-菜单映射**
- **5 座桥梁**：赣江一桥、鄱阳湖大桥、东郭虎渠桥、雪山高架桥、松花江北桥（覆盖 5 个气候带）
- **每桥 4 条 AADT 历史**
- **11 类设备**：拉索、伸缩缝、支座、桥面铺装、防撞栏、监测传感器等
- **38 台设备**，按桥梁分布，状态多样
- **大量历史数据**：36 个月保养记录、按设备年龄随机生成的故障工单
- **5 条采购订单**（不同状态）
- **每台设备的生命周期事件链**：purchase → commission → 历次 maintain/repair → (scrap)

## 🔧 常用查询示例

```javascript
// 查所有使用中的设备
const devices = await dbApi.query('device_archive', d => d.status === 'in_use');

// 查指定设备的所有生命周期事件
const events = await dbApi.query('lifecycle_event', e => e.device_id === 'xxx');

// 查 30 天内的故障工单
const cutoff = new Date(Date.now() - 30*86400*1000).toISOString();
const orders = await dbApi.query('fault_order', o => o.report_date >= cutoff);

// 系统配置读取
const cfg = await dbApi.get('sys_config', 'repair_residual_threshold');
```

## 🔄 数据库重置

如需清空数据重新初始化：

**方式一**：浏览器开发者工具 → Application → Storage → Clear site data

**方式二**：浏览器控制台执行

```javascript
indexedDB.deleteDatabase('bridge_lifecycle_db');
location.reload();
```

## 🚀 迁移到生产数据库

如果将本系统迁移到 Spring Boot + MySQL 真实后端，对应关系为：

| IndexedDB ObjectStore | MySQL 表 | 引擎 |
|---------------------|---------|------|
| 17 个 ObjectStore | 17 张 InnoDB 表 | UTF-8MB4 |
| 主键 string id | varchar(36) PK | UUID |
| 唯一索引 | UNIQUE KEY | 同名字段 |
| 时间字段 | datetime | 默认 CURRENT_TIMESTAMP |

只需替换 `dbApi.*` 为 RESTful Axios 调用即可，业务逻辑无需重写。
